# Support

## How to file issues and get help

### GitHub issues

We use [GitHub Issues](https://github.com/microsoft/hyperscale-db-sdk-for-java/issues/new/choose) to track bugs, questions, and feature requests.
GitHub issues are free, but **response time is not guaranteed.**

To keep the issue queue focused and manageable, issues that are over two years old and have not been updated in the last 30 days may be automatically closed.

### Azure support tickets

Customers with an [Azure support plan](https://azure.microsoft.com/support/options/) can open an [Azure support ticket](https://azure.microsoft.com/support/create-ticket/).
**We recommend this option if your problem requires immediate attention.**

### Community resources

- Ask a question on [Microsoft Q&A](https://learn.microsoft.com/answers/products/azure) and tag it with the relevant Azure service (e.g. `azure-cosmos-db`, `amazon-dynamodb-on-azure`, `google-cloud-spanner-on-azure`).
- Ask a question on [Stack Overflow](https://stackoverflow.com/questions/tagged/azure) and tag it with `azure`.
- Search for similar issues in the [GitHub repository](https://github.com/microsoft/hyperscale-db-sdk-for-java/issues).
- Take a look at the [Azure SDK blog](https://devblogs.microsoft.com/azure-sdk/).

### Security issues

Security vulnerabilities should be reported privately via the GitHub
[Security Advisory](https://github.com/microsoft/hyperscale-db-sdk-for-java/security/advisories/new) page,
or by email to [secure@microsoft.com](mailto:secure@microsoft.com).
**Do not open a public GitHub issue for security vulnerabilities.**
You should receive a response within 24 hours. Further information, including the MSRC PGP key, can be found in
the [Security TechCenter](https://www.microsoft.com/msrc/faqs-report-an-issue).

See also [SECURITY.md](./SECURITY.md) for the full security policy.
